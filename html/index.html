Core topics

Data Communication Concepts

Data communication is the exchange of digital or analog information between devices through wired or wireless media. It depends on signals, protocols, addressing, timing, and reliability.

01

Sender

The device that creates the message, such as a computer, phone, server, sensor, or telegraph key.

02

Message

The data being transmitted. It can be text, images, sound, video, commands, or machine readings.

03

Medium

The path used to carry the signal, including copper cables, fiber optics, radio waves, and satellites.

04

Receiver

The device that accepts the signal, decodes it, checks it, and presents the message to the user or system.

05

Protocol

The rules that control formatting, addressing, sequencing, error checking, and delivery confirmation.

06

Noise

Unwanted interference that can distort a signal and cause errors, delay, or loss of information.

Transmission Modes

Simplex, half-duplex, and full-duplex describe the direction of communication.

Transmission Modes

Simplex sends data one way only, such as a keyboard to a computer. Half-duplex allows both devices to communicate, but one at a time. Full-duplex allows both sides to send and receive together, like a phone call.

Analog and Digital Signals

Signals carry data through electrical, optical, or wireless patterns.

Analog and Digital Signals

Analog signals are continuous waves, like traditional radio. Digital signals use discrete values, usually binary 0s and 1s. Digital communication is easier to store, process, compress, encrypt, and correct when errors happen.

Bandwidth and Speed

Bandwidth affects how much data can pass through a channel.

Bandwidth and Speed

Bandwidth is the capacity of a communication channel, often measured in bits per second. Real speed also depends on distance, congestion, signal quality, hardware, and delay. More bandwidth usually improves performance.

Transmission Media

Media are the physical or wireless paths that carry signals.

Transmission Media

Guided media include twisted pair, coaxial, and fiber optic cable. Unguided media include radio waves, microwave, infrared, Bluetooth, Wi-Fi, and satellite links. Each medium has different cost, range, speed, and reliability.

Error Detection

Communication systems must detect mistakes in transmitted data.

Error Detection

Noise and interference can damage transmitted bits. Parity checks, checksums, and CRC help detect errors. If an error is found, the receiver can reject the data and request retransmission for accuracy.

Multiplexing

Multiplexing lets many signals share one communication channel.

Multiplexing

Multiplexing combines multiple data streams so they can travel through one medium. It may divide the channel by time, frequency, wavelength, or code. This makes communication more efficient and reduces cost.

Evolution of Data Communication

Data communication developed from simple signals into global digital networks.

Evolution of Data Communication

Early communication used smoke signals, drums, letters, and messengers. The telegraph introduced electrical signals over long distances. Telephones added voice transmission, radio enabled wireless broadcasting, and computers created digital networking. Today, fiber optics, satellites, mobile networks, cloud systems, and the internet allow fast global communication.

Global Internet

OSI and TCP/IP

Network Models Explained

Network models organize communication into layers. Each layer has a specific job, making complex systems easier to design, understand, and troubleshoot.

OSI Model: 7 Layers

  1. Physical: Sends raw bits through cables, radio waves, connectors, and electrical or optical signals.
  2. Data Link: Handles local delivery, MAC addresses, frames, and error detection on the same network.
  3. Network: Routes packets between networks using logical addresses such as IP addresses.
  4. Transport: Provides end-to-end delivery, segmentation, reliability, and flow control using TCP or UDP.
  5. Session: Starts, manages, and ends communication sessions between applications.
  6. Presentation: Converts data formats, compresses data, and supports encryption or decryption.
  7. Application: Provides user-facing network services such as web browsing, email, file transfer, and DNS.

TCP/IP Model: 4 Layers

  1. Network Access: Controls physical transmission and local network delivery through Ethernet, Wi-Fi, or other media.
  2. Internet: Uses IP addressing and routing to move packets across different networks.
  3. Transport: Uses TCP for reliable ordered delivery or UDP for faster connectionless delivery.
  4. Application: Contains protocols such as HTTP, HTTPS, DNS, SMTP, FTP, and DHCP.

The TCP/IP model is the practical model used by the internet. It focuses on real communication between devices and combines several OSI functions into fewer layers.

Cryptography

Cryptography protects communication from unauthorized access and tampering.

Cryptography

Cryptography converts readable data into ciphertext so only authorized users can understand it. Encryption protects privacy, hashing checks whether data has changed, and digital signatures confirm the sender. It is used in HTTPS, Wi-Fi security, VPNs, banking, messaging, and secure email.

Click to flip

Pretty Good Privacy

PGP is used to protect messages, files, and email communication.

Pretty Good Privacy

Pretty Good Privacy, or PGP, uses public-key cryptography to secure communication. A sender encrypts data with the receiver's public key, and the receiver opens it with a private key. PGP can also sign messages to prove authenticity and protect against message changes.

Secure Data Communication

Secure communication protects confidentiality, integrity, and availability.

Secure Data Communication

Secure data communication means information is protected while moving between devices. It uses encryption, authentication, firewalls, VPNs, access control, secure protocols, backups, and monitoring. Good security prevents data theft, impersonation, data modification, and service interruption.

Satellite Communication

Satellites support wide-area communication across long distances.

Satellite Communication

Satellite communication sends signals from an earth station to a satellite and back to another location. It supports TV, GPS, aviation, ships, remote communities, disaster recovery, and global internet. Satellites cover wide areas but may face delay, weather interference, and high setup cost.

Modulation

Modulation prepares signals for efficient long-distance travel.

Modulation

Modulation changes a carrier wave so it can carry information. It may change amplitude, frequency, or phase. This helps signals travel farther, resist interference, and share communication channels efficiently.

Network Devices

How Telegraph Communication Works

The telegraph is one of the earliest electronic data communication systems. It used electrical pulses to send coded messages over long distances.

From key press to message

A telegraph operator presses a key to close an electrical circuit. When the circuit closes, current flows through the telegraph line to a distant station. At the receiver, the current activates an electromagnet that produces clicks or marks.

Short pulses represent dots and long pulses represent dashes. These dots and dashes form Morse code, which is translated into letters, words, and full messages. This made long-distance communication much faster than sending physical letters.

  • Telegraph key opens and closes the circuit.
  • Electrical pulses travel through the wire.
  • Morse code converts pulse patterns into readable messages.
